## 잔여 위험 / 후속 과제

- 이 환경에는 Redis, web node_modules, Python `redis` package가 없어 `/project-view` end-to-end 기동은 실행하지 않았다. 명세상 설치 명령은 실행 금지라 check-deps의 안내/exit 보고까지만 검증했다.
- `start.sh`의 proxy active 판정은 vendored `cmux-proxy.sh status`의 한국어 출력 포맷에 의존한다. vendor 재동기화 때 status 문자열 계약이 바뀌면 launcher도 같이 확인해야 한다.
- `stop.sh`는 launcher-owned root PID를 종료한다. `npm run dev` 하위 프로세스가 SIGTERM을 무시하는 환경에서는 후속 port check가 남은 listener를 잡아낼 것이다.
- 현재 작업 디렉토리는 `.git` 저장소가 아니어서 git diff/status 기반 변경 요약은 만들 수 없었다.

## 다음 코드 리뷰 권장 영역

- `/project-view` end-to-end: 실제 cmux/Redis/node_modules/Python deps가 준비된 머신에서 proxy inject, READY URL, terminal polling까지 확인.
- `db.sh migrate`: 0.1.0 실사용 DB 샘플을 대상으로 데이터 보존과 FK cascade 동작 확인.

- re-vendor contract: `cmux-proxy.sh status` 출력과 runtime web port 계약을 자동 검사하는 lightweight fixture 추가.
